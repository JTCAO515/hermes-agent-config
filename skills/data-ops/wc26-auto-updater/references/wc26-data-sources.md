# WC26 Data Sources & Verification

## Data Fields & Their Sources

| Field | Source | Freshness | Verification Method |
|-------|--------|-----------|-------------------|
| Match results | [FIFA.com scores & fixtures](https://www.fifa.com/en/tournaments/mens/worldcup/canadamexicousa2026/scores-fixtures) | Real-time (live) | Direct extract from FIFA page |
| FIFA rankings | [ESPN](https://www.espn.com/soccer/story/_/id/46664763/fifa-mens-top-50-world-rankings) / [SofaScore](https://www.sofascore.com/football/rankings/fifa) / [Wikipedia](https://en.wikipedia.org/wiki/FIFA_Men%27s_World_Ranking) | Monthly | Cross-reference 2+ sources |
| Team attack/defense ratings | Manually assigned in `data/wc2026_teams.json` | Static (needs update) | Compare against FBref xG, Understat |
| Group composition | [FIFA teams page](https://www.fifa.com/en/tournaments/mens/worldcup/canadamexicousa2026/teams) | Static (draw done) | Confirm all 12 groups × 4 teams |
| Polymarket odds | Static JSON snapshot in `data/polymarket_odds.json` | Needs real-time API | Direct Polymarket API call |
| Knockout bracket | Generated by `data/generate_wc2026.py` | After group results update | Verify bracket structure vs FIFA |

## Scraping Real Match Results

Use `web_extract` on the FIFA scores page. Key data points per match:
- Date, Home team, Score (home-away), Away team, Stage, Venue

The FIFA page shows completed matches with FT (Full Time) scores and upcoming matches with kickoff times.

## Data Correction Script

`data/correct_data.py` is the canonical correction toolkit. It:
- Standardizes all team names to FIFA official names
- Recalculates attack/defense ratings from real FIFA rankings
- Fixes team names in matches JSON
- Can be re-run after FIFA ranking updates

## Auto-Updater Logic (Flow Decision)

```
Real match exists on FIFA.com?
  ├─ Yes → Record real result (do NOT simulate)
  ├─ No, but kickoff time is past → Use update_results.py --days 0 (check only)
  └─ No, future match → Leave as prediction (no action needed)
```
