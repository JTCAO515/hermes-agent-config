# Assumptions and Prioritization

Identify what's uncertain, then decide what to work on first.

## A. Identify Assumptions (Existing Product)

For a feature idea in an existing product: list assumptions across desirability (will users want it?), viability (can we build/support it?), and feasibility (is the tech/infra ready?). Classify each as low/medium/high risk.

## B. Identify Assumptions (New Product)

For a new product idea: broader assumption set covering market need, willingness to pay, distribution channels, competitive response, technology readiness, and regulatory risk. Focus on the assumptions that, if wrong, kill the concept.

## C. Prioritize Assumptions

Use an Impact x Risk matrix: plot each assumption on a 2x2 grid (Impact: how much damage if wrong? Risk: how likely is it wrong?). Test high-impact + high-risk assumptions first. Output: ordered test queue.

## D. Prioritize Features

Prioritize a backlog of feature ideas. Use the right framework for the context:
- **RICE** (Reach x Impact x Confidence / Effort): data-driven, for well-understood problems
- **Value vs. Complexity**: 2x2 matrix, for directional decisions
- **Kano Model**: basic needs vs performance vs delighters, for user satisfaction
- **MoSCoW**: Must-have / Should-have / Could-have / Won't-have, for scope management

Output: prioritized feature list with rationale and recommended framework.
