# 跨Agent个人画像示例

> 当用户要求"写我的memory给别的AI agent用"时，以此格式输出。
> 核心原则：写**人的工作方式**，不是写项目详情。

## 示例（曹金涛/JT）

```
# 曹金涛 (JT) — AI Agent 合作指南

## 基础信息
中文名曹金涛，英文名JT。1998年，广州。UKentucky本科+GWU市场营销硕。
TOEFL108，粤语。English-native所有前端界面。

## 核心沟通纪律

**指令解码：**
- "继续"=直接做下一步，不要等确认
- "开始"=确认了直接跑
- "两个一起写"=并行执行
- "为什么卡住了"=给[原因+修复方案]，不要长篇日志
- "检查原因"+链接=诊断+修复，不只诊断

**节奏：** 我分析→给方案→拍板→批量执行。不等长篇确认。
**汇报：** Git push后必须微信通知。

## 不喜欢
- 长篇分析/啰嗦回答/连环追问
- GUI操作教学（优先命令行方案）
- 技术术语解释（他是业务决策者）

## 技术定位
非技术背景但视野广：会读Vercel构建日志、开安全组。
Windows本地开发，命令行 > GUI。

## 项目敏感点
数据必须真实，预测可模拟但须可推敲。
EV一位小数(.toFixed(1))。所有UI英文。
```

## 写作检查清单

- [ ] 指令解码（用户说X意味着AI该做什么？）
- [ ] 沟通节奏（先分析→给方案→拍板→执行？还是直接干？）
- [ ] 汇报纪律（有没有must-notify渠道？）
- [ ] 不喜欢什么（长篇？追问？GUI？术语？）
- [ ] 技术定位（多懂？需要什么level的解释？）
- [ ] 项目信息一句带过即可，不要展开架构细节
