# 示例：VP-Hermes (VisePanda) Brainstorming 产出

> 产品：AI China Travel Platform (go2china.space)，36-city 知识库 + SSE 聊天
> 使用模式：A（现有产品创意）+ 跨视角 PM/Designer/Engineer

## 产出结构范例

### PM 视角（5 个想法）
| # | 功能 | 一句话价值 |
|---|------|-----------|
| 1 | 行程预算估算器 | 消除"去中国贵不贵"心理门槛 → 向上销售 |
| 2 | 行程导出/分享 | 可携带可协作 → 病毒传播入口 |
| 3 | 城市对比模式 | 降低决策焦虑 → 更高参与度 |
| 4 | 酒店/民宿推荐 | 一站式闭环 → 联盟佣金 |
| 5 | 一键签证材料包 | 解决核心痛点 → 高价值转化 |

### Designer 视角（5 个想法）
| # | 功能 | 体验升级点 |
|---|------|-----------|
| 1 | 行程时间线可视化 | 从"读段落"到"看一眼" |
| 2 | 城市卡片滑动探索 | Tinder 式交互 |
| 3 | 熊猫导游表情 | 品牌人格化 |
| 4 | 沉浸式目的地预览 | "去之前先感受" |
| 5 | 行程步骤向导 | 降低聊天空白焦虑 |

### Engineer 视角（5 个想法）
| # | 功能 | 技术亮点 | 成本 |
|---|------|---------|------|
| 1 | 行程后端持久化 | localStorage → Postgres | 中 |
| 2 | 多语言支持 | 单 prompt 参数切换 | 低 |
| 3 | PWA 离线模式 | Service Worker 缓存 | 中 |
| 4 | AI 生图集成 | 用已有 Seedream API | 低 |
| 5 | 热度排序 | 数据驱动推荐 | 低 |

### 优先级排序（5 选 TOP）
排序依据：战略对齐 × 用户影响 × 技术可行性 × 差异化

| 排名 | 功能 | 视角 | 理由 |
|------|------|------|------|
| 🥇 | 行程时间线可视化 | Designer | 最大体验升级，低代码，高感知 |
| 🥈 | 行程后端持久化 | Engineer | 基础设施，必先建 |
| 🥉 | 城市对比模式 | PM | 核心决策场景 |
| 4th | 熊猫导游表情 | Designer | 低成本高回报 |
| 5th | 签证材料包 MVP | PM | 壁垒级但需精细范围控制 |

## 从 Brainstorming 到 Plan

1. **Grill** — 用 grill-with-docs 压力测试（这个例子揭示了 5 个关键风险：冷启动、解析稳定性、数据完整性、范围陷阱、依赖顺序）
2. **PLAN.md** — 正式路线图，每批含 scope/out-of-scope/工时/版本/依赖
3. **ADRs** — 每个关键决策一条（如 persistence 策略、对比容错、签证 MVP 范围）
4. **版本推进** — 按 B1a→B1b→B2→... 顺序执行
