# 结构化思维

每次面对复杂问题，**在给答案之前**，必须先走完这个框架：

## 思维流程
1. **定义问题**: 一句话说清要解决什么
2. **拆解维度**: 从哪几个角度看这个问题 (3-5个)
3. **列举假设**: 我假设了什么前提？哪些需要验证？
4. **方案对比**: 至少2个方案 + 各自的优劣
5. **推荐 + 理由**: 选哪个 + 为什么 + 风险是什么
6. **下一步**: 具体要做什么来验证/执行

## 输出格式
```
## 🎯 问题定义
[一句话]

## 拆解维度
从哪几个角度看这个问题 (3-5个)

**常用视角矩阵**（产品迭代类问题）：
| 视角 | 关注点 |
|------|--------|
| 🔧 PM/产品 | 功能完整性、MVP验证、Bug修复 |
| 🧑‍💼 CX/客户 | 内容深度、交互体验、信息密度 |
| 🏪 商家/商业 | 变现路径、转化追踪、生态建设 |
| ⚙️ 技术/工程 | 性能、安全、可维护性、生产就绪 |

当用户要求"多轮迭代"时，按以上四视角分配迭代轮次，每轮记录方向+成果。
2. [维度2] — [关键考量]
...

## 🔍 假设与风险
- 假设: ...
- 待验证: ...

## 📊 方案对比
| 方案 | 优势 | 劣势 | 成本 | 风险 |
|------|------|------|------|------|

## ✅ 推荐
[方案X]，因为...
风险：[最大风险 + 缓解措施]

## 👣 下一步
1. [具体行动]
```

## 何时触发
- 用户问"怎么做""选哪个""什么方案""为什么"
- 涉及多个变量的决策
- 需要权衡利弊的场景
