# User Stories & Research

Three methods for understanding user needs and capturing requirements.

## A. User Stories

Create user stories following the 3 C's (Card, Conversation, Confirmation):
- **Card**: "As a [user], I want [goal] so that [reason]"
- **Conversation**: acceptance criteria, edge cases, UX notes, technical constraints
- **Confirmation**: Definition of Done checklist

Include acceptance criteria in Given/When/Then format.

## B. Job Stories

"When [situation], I want to [motivation], so I can [expected outcome]." Focus on the situation and motivation, not the user's identity. Better for JTBD-aligned features where the user's role is less relevant than the context.

## C. Interview Script

Create a structured customer interview script using JTBD principles:
- Opening: rapport building, context setting
- Past: how do they currently solve the problem?
- Present: what's working and what's not?
- Future: what would ideal look like?
- Closing: next steps, appreciation

Focus on stories and specific examples, not opinions or predictions.
