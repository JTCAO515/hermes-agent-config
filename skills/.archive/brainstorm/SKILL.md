# 结构化头脑风暴

当用户需要想法、创意、方案时，不要直接给答案。先结构化发散，再收敛归纳。

## 触发条件

用户说"帮我想想""有什么思路""还有什么方向""怎么优化""给几个方案"时，启动脑暴流程。

## 流程

### 阶段1：发散（不评判，只生成）

根据问题类型选择框架：

| 问题类型 | 框架 |
|----------|------|
| 产品功能 | SCAMPER（替代/组合/调适/修改/他用/消除/重排）|
| 市场策略 | 4P+AI（产品/价格/渠道/推广+技术） |
| 内容创意 | 6角度（痛点/情绪/反常识/数据/故事/对比） |
| 体验优化 | 用户旅程图（前/中/后 × 触点） |
| 商业模式 | 画布九宫格 |

**规则**：每个角度至少产出3个想法，先不管可行性。

### 阶段2：收敛（筛选+排序）

用2×2矩阵快速收敛：
- 纵轴：影响力（高/低）
- 横轴：实现难度（易/难）

优先推荐"高影响+易实现"象限的想法。

### 阶段3：精炼（挑3-5个最佳）

对入选想法做：
- 一句话描述
- 为什么选它
- 下一步行动（验证什么）

## 输出格式

```
## 🧠 脑暴主题：XXX

### 发散（N个角度 × M个想法）
| # | 角度 | 想法 | 来源 |
|---|------|------|------|
| 1 | ... | ... | SCAMPER-S |

### 收敛（2×2矩阵）
高影响·易实现 → ⭐ 优先
高影响·难实现 → 📋 规划
低影响·易实现 → 🏃 速赢
低影响·难实现 → 🗑️ 暂缓

### 精炼（TOP 3-5）
1. **[想法名]** — 一句话
   - 为什么：xxx
   - 下一步：验证xxx
```

## 领域适配

如果用户有特定领域（跨境/旅游/产品），自动注入行业参数：

### 旅游行程产品
- 发散角度：客群分层（奢华/家庭/情侣/独旅/银发）· 季节特性 · 差异化体验 · 打包方式 · 渠道策略
- 收敛依据：客单价提升潜力 vs 执行复杂度

### 跨境网络/SaaS产品
- 发散角度：获客渠道 · 留存机制 · 定价模型 · 竞品差异 · 新场景拓展
- 收敛依据：用户价值 vs 开发成本 + 合规风险

## 脑暴后行动

每次脑暴结束后，询问用户：
- 把最佳想法落地为具体方案？
- 存入 skills 作为后续参考？
- 需要进一步细化某个方向？
