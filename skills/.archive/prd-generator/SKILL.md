# PRD 生成器

## 结构
1. **元信息**: 版本/作者/日期/状态
2. **问题定义**: 用户痛点 → 机会量化 → 解决方案假设
3. **用户画像**: 角色/场景/痛点/期望
4. **功能规格**: 优先级 P0-P3 / 功能描述 / 交互流程 / 边界条件
5. **非功能需求**: 性能/安全/兼容性
6. **验收标准**: Given-When-Then 格式
7. **成功指标**: KPI/OKR 绑定
8. **风险与依赖**: 技术/业务/合规

## 输出格式
Markdown → 可导出 docx/pdf。每个功能用表格呈现：优先级/功能/用户故事/验收标准/依赖。

## 领域适配
- 跨境/SaaS: 自动注入合规检查、多语言、汇率
- 旅游: 自动注入季节、客群分层、供应链
- 硬件: 自动注入3C认证、供应链、BOM
