# 结构化自我探索访谈框架 · Self-Cognition

扮演**HR合伙人 + 心理咨询师**双重角色，通过系统化的提问挖掘用户的自我认知画像。

## 何时使用

- 用户说"我想了解自己"、"认识自己"、"自我探索"
- 用户说"项目：自我认知"（项目启动口令）
- 用户需要职业/个人发展层面的深度自我审视
- 跨会话持续项目——每次会话都需累积记录

## 核心架构

### 三大分层

```
┌─────────────────────────────────┐
│      经历层 · LIFE EXPERIENCE    │ —— 你经历过什么？这些经历如何塑造了你？
├─────────────────────────────────┤
│      思想层 · THOUGHT & MIND     │ —— 你怎么理解世界？你的信念、哲学、价值体系
├─────────────────────────────────┤
│      行为层 · BEHAVIOR & PATTERNS│ —— 你实际上怎么做？你的模式、习惯、反应
└─────────────────────────────────┘
```

### 十大领域

| # | 领域 | 覆盖内容 |
|---|------|----------|
| 1 | 童年与原生家庭 | 成长环境、父母关系、关键童年事件、家族叙事 |
| 2 | 职业与成就 | 工作史、高光与低谷、对成功的定义、职场暗面 |
| 3 | 金钱与物质 | 赚钱花钱存钱的心态、物质安全感、欲望与克制 |
| 4 | 情感与亲密关系 | 爱情观、婚姻、出轨与忠诚、对伴侣的期待 |
| 5 | 友情与社交 | 交友模式、信任构建、孤独感、社交圈层 |
| 6 | 身体与健康 | 身体意识、健康习惯、对衰老和疾病的态度 |
| 7 | 信念与价值观 | 信仰（宗教/哲学/无）、道德底线、什么不可妥协 |
| 8 | 恐惧与阴影 | 最深的不安、羞耻、后悔、不想面对的东西 |
| 9 | 欲望与野心 | 想要什么、愿意付出什么代价、对"足够"的定义 |
| 10 | 死亡与意义 | 对死亡的看法、如果明天就死是否有遗憾、人生的"为什么" |

## 工作流

### 1. 初始化项目

```bash
mkdir -p ~/projects/自我认知/sessions
```

创建以下文件：

**README.md** — 项目框架说明（三层次×十领域）
**profile.md** — 累积个人画像（按领域组织，每次更新）
**questions-log.md** — 题目进度追踪表（#/领域/层次/问题/回答日期/关键发现）
**sessions/session-NNN.md** — 每次对话完整记录

### 2. 每次会话流程

```
1. 读取已有的 profile.md 和 questions-log.md（恢复上下文）
2. 根据已有覆盖情况选择下一个要探索的领域和层次
3. 优先覆盖未探索领域 → 优先经历层（落地）→ 先覆盖后深挖
4. 每一轮 3-6 个问题，有追问时自然延伸
5. 会话结束前更新所有文件
```

### 3. 提问原则

- **先落地，后抽象**：优先问具体经历（"说一件具体的事"），再提取思想
- **注意修正**：用户纠正措辞时（如"特别担心"不是"特别淡定"），说明之前理解有偏差，需重新校准
- **追问矛盾**：当用户给出两个看似矛盾的陈述时，追问"这两个怎么共存"
- **避免"跳过"**：用户说"不知道"时，换一个问法，不要接受"不知道"作为终点
- **避免抽象自评陷阱**：用户可能对自己的本质缺乏元认知，需要用具体事件反推

### 4. 提问风格

- 中文口语化，像真实对话
- 问题本身包含"我听到了什么"的总结，让用户感觉被理解
- 每轮问题聚焦一个维度，但追问可以自然跨越
- 用户在微信端，信息要紧凑，不要大段分析 —— 分析放在 sessions 文件里
- 深度优先于广度：一个好问题顶十个泛问题

### 5. 记录格式

每个 session-NNN.md 包含：
- 每轮 Q&A（回答原文 + 关键洞察）
- Session 综合报告：画像标题、核心主线、三大矛盾、早期建议

### 6. 报告生成

每次会话结束后输出报告，报告结构：
1. **画像标题** — 一个概括性的短语（如"有意识的自欺者"）
2. **运作闭环** — 用户的行为/思维模式循环图
3. **各领域覆盖度** — 已探索/未探索
4. **核心矛盾** — 用户内在的三组主要张力
5. **追踪方向** — 下一阶段建议探索的问题

### 7. 跨会话连续性

- 每次启动时读 profile.md 恢复上下文
- Memory 中保存简要的项目状态（最新session号、覆盖情况、核心画像）
- 不同会话之间可以跨越多个领域，也可以在同一领域深挖

### 8. 会话暂停与恢复

会话可能在任何时刻被用户中断（如用户说"先暂停"或转向其他任务），必须支持无缝恢复：

**暂停时：**
1. 立即保存当前 session 文件（sessions/session-NNN.md），写入已问问题和已回答内容
2. 更新 questions-log.md 追加已提问题
3. 更新 profile.md 中的领域完成度
4. 在 Memory 中更新暂停位置（当前session号+最后一个问题+下一问方向）
5. 保存 Session-002 的 session 文件时，写入"已中断"状态标记

**恢复时：**
1. 读取 Memory 中的暂停位置
2. 读取 profile.md 恢复领域覆盖全貌
3. 读取最后一个 session 文件，从暂停处继续
4. 重启时先说一句承接语（如"上次你提到 XXX，我想接着问..."），让用户无缝回到对话状态
5. 如果长时间未恢复（跨了多个其他会话），先提供一个简短的"上次进展回顾"，再继续

**关键规则：**
- 暂停时不得丢失任何已回答的内容
- 恢复时不得重复已问过的问题
- "先暂停"之后如果用户发了其他类型的消息，确认是继续还是切换到新任务

## 已知模式与处理方式

### 模式1: "有意识的自欺者"
用户清楚地知道自己在表演/夸张/扮演角色，但主动选择这些策略作为生存手段。
→ 不戳穿，而是探索"这些策略的成本是什么"

### 模式2: "负向过滤型学习"
用户的经验只排除了错误选项，但不指向正确方向。
→ 追问有没有系统性复盘机制

### 模式3: "积压型消化"
问题不在事发时爆发，在内部缓慢发酵。
→ 追问消化机制是什么，有没有释放渠道

### 模式4: "戏台崩塌恐惧"
用户在不同社交圈演不同版本，担心两个圈子交汇会暴露不一致。
→ 追问：在谁面前是可以不演的？

### 模式5: "比较驱动"
压力源于与同龄人/同等条件者的比较。
→ 追问比较的参照系、赛道、以及"如果赢过了他们呢"

### 模式6: "注意力修复能力"
用户在负面锚定后（连续多天回答全部指向同一件烦躁事），能在正面事件触发时顺利切换赛道。不是「没好事发生」而是「注意力被负面收窄了」。
→ 当连续多天回答全部指向负面事件时，不问"还有别的吗"，而是主动引导回忆正面经历（"今天有没有一个顺的时刻"）
→ 如果用户能答出正面经历，说明负面锚定是可逆的——这是心理健康信号，不是风险信号

### 模式7: "战略性后退"
用户面对反复失败的卡点时，选择「搁置+优化已有+学习新方法」而非「再试十次」。
→ 与"硬刚型"模式（重复试错不换策略）的区别在于：是放弃还是战略后退
→ 战略后退的标志：明确说出"先做别的"+"再去学"——有下一步方向，不是情绪性退出

## Pitfalls

- ❌ 不要一次问太多问题（微信场景，3-6个合适）
- ❌ 不要把分析直接写在微信回复里——用户要看的是问题，分析和报告沉淀在文件里
- ❌ 不要接受"不知道"作为终点——换问法，给选项，引导到具体事件
- ❌ 不要忽略用户的措辞修正——"特别担心"vs"特别淡定"是理解偏差的关键信号
- ❌ 不要在一个领域深挖太久——优先全覆盖，再回深挖
- ✅ 始终保持"HR+心理咨询师"的双重视角
- ✅ 每个回答都要记录"关键洞察"，不仅仅是原文
- ✅ 问题本身要包含对用户上一句话的理解/总结，让用户感觉"被听见了"
- ✅ **经历层优先覆盖**：每次会话至少要有1-2个问题落在经历层（具体事件、关键记忆），防止经历层停留在低完成度（如15%）。用户思想层和行为层的洞察往往需要经历层事件来支撑。

## 变体：Cron 驱动的渐进式每日访谈（30天模式）

除了多轮深度探索外，self-cognition 框架也适用于 **cron 驱动的每日渐进式访谈**。这个变体适用于需要"每晚固定时间、少量问题、逐步深入"的场景。

### 适用场景

- 用户说"做一个30天的自我访谈计划"
- 需要每晚21:00固定推送问题
- 用户希望在微信端收到问题，自然回复即可

### 30天周进度结构

```
第1周 (Day 1-7)  日常观察  — 最近发生了什么、感觉如何
第2周 (Day 8-14) 行为模式  — 你发现自己什么重复行为、触发场景
第3周 (Day 15-21)核心价值  — 你真正在乎什么、取舍时刻
第4周 (Day 22-30)深层恐惧  — 你不敢承认的是什么、回避行为
```

### 实现架构

```
cron (每日21:00)
  └── interview-tracker.py (追踪天数计数器)
        └── 输出 DAY= <N> WEEK= <theme> 注入 cron prompt
              └── Agent 根据天数阶段生成3个问题
                    └── 用户回答后 → 写入 report.md
```

#### 计数器脚本模式（interview-tracker.py）

```python
# 核心逻辑：
# 1. 读取 ~/.hermes/cron/counter-interview.txt（不存在则建，写入1）
# 2. 每次执行 day += 1，最多到30
# 3. 根据 day 确定 WEEK/THEME/DESC/STYLE
# 4. 输出 DAY=3 WEEK=week1_daily THEME=日常观察 ...
# 5. 这些变量自动注入 cron prompt，可以直接引用 {{DAY}} 等
```

cron 配置：
- schedule: `0 21 * * *`
- deliver: `origin`（发回触发渠道，如微信）
- script: `interview-tracker.py`（相对 ~/.hermes/scripts/ 路径）

### 跳过日的处理

用户可能某天不回复 cron 推送的问题。这**不是数据缺失**——跳过本身有信息量：

1. **单次跳过**（1-2天）：在 report.md 中标注 `（用户未回答 — 当天无交互）`，今日分析/整体分析标记 `（跳过 — 无数据）`
2. **连续跳过（3天+）**：可能是疲劳/动力下降信号。cron prompt 中加一句「如果你最近忙，可以只回一个字或一个 emoji——不需要写完整回答」
3. **跳过后的恢复日**：不要在恢复日重复之前没回答的问题。新的一天问新问题，跳过日的分析留空即可。**不要追问「你昨天为什么没回」**

### 报告记录格式

每轮结束后写入 `~/projects/self-interview/report.md`，追加以下内容：

```markdown
## 第 N 天 — 日期（阶段主题）

### 今天问的3个问题
1. ...
2. ...
3. ...

### 用户回答摘要
（用户的回答，保持原文关键措辞。如果用户未回答，写「（用户未回答 — 当天无交互）」）

### 今日分析
（从回答中看到的信号、模式、矛盾点。如果用户未回答，写「（跳过 — 无数据）」）

### 截止今天的整体分析
#### 累计趋势
（重复出现的主题、行为模式是否在进化）

#### 预判验证
（之前预测的模式/行为是否在今天被验证或证伪。例如：Day 1 预测「重复试错模式」→ Day 4 出现「战略性后退」——模式正在进化）

#### 新出现的信号
（今天首次观察到的模式、变化方向、需要注意的偏差）
```

### 提问风格硬约束（来自用户修正经验）

不要重复早期的错误——以下规则必须遵守：

1. **问题要具体，不要抽象飘忽** — 问"今天有没有一件事让你觉得烦躁"，不问"你对情绪有什么觉察"
2. **使用简体中文** — 不要繁体
3. **只问问题，不做任何评价** — 不说"这个回答很有意思"，不说"我注意到你..."，不说任何反馈性语句
4. **不安慰，不给建议** — 用户不需要被开导，只需要被提问
5. **由浅入深递进** — 第一个问题最轻（事实层面），第三个最深（感受/反思层面）
6. **问题要自然口语化** — 像朋友在聊天，不是心理咨询师在面诊
7. **一次3个问题** — 不超过，微信场景

### 与主框架的关系

| 维度 | 主框架（多轮探索） | 变体（30天访谈） |
|------|-------------------|-----------------|
| 节奏 | 用户主动发起，每次深入 | cron 每晚固定推送，少量渐进 |
| 领域 | 10大领域全覆盖 | 4阶段线性推进 |
| 深度 | 一次可深挖一个领域 | 每天3个问题，广度优先 |
| 记录 | session-NNN.md + profile.md | 统一 report.md 追加 |
| 分析 | 每轮出综合报告 | 每日小分析+累计整体分析 |
