# 自我认知项目 · 实战录

## 项目概况

由 guided-interview 技能驱动的"自我认知"项目，面向 HR + 心理咨询师双角色。首次运行完成 8 轮问答（Q1-Q8），延续会话完成 22 轮总问答（Q1-Q22），覆盖 6/10 领域，产出完整底层系统画像。

---

## 第一轮会话（Q1-Q8）

### 维度轨迹

```
自我定义 ─→ 追问矛盾 ─→ 压力来源 ─→ 情绪管理 ─→ 决策风格 ─→ 决策代价 ─→ 自我评价 ─→ 他人视角
  (Q1)        (Q2)        (Q3)        (Q4)        (Q5)        (Q6)        (Q7)        (Q8)
```

### 发现的高价值提问技巧

#### 1. "抓矛盾"追问法
当用户给出看似矛盾的答案时（如"父亲"vs"地球online唯一主角"），直接点出矛盾并追问共存机制。

**效果**：迫使用户自我剖析，能挖出深层信念。本例产出了核心价值观陈述（"我知道我是过客，但必须扮演主角"）。

#### 2. "机制代价"追问
在用户描述完一个行为模式后（如"凭感觉决定+事后找理由说服自己"），跟进问"这套机制有没有翻过车？"

**效果**：让用户从自我辩护模式切换到反思模式，暴露盲区和脆弱点。

#### 3. 跨答案连接
将用户不同问题的回答连接成一条主线（"你的决策方式和你扮演主角是同一种机制"），让用户看到自己的系统性模式而非孤立行为。

**效果**：产生"aha moment"效应，用户自动开始更深层次的自我认知。

#### 4. 维度切换节奏
- 同一个维度问 3-4 轮（含初始问题 + 1-2 次追问）后换维度
- 追问方向：表层回答 → 底层动机 → 认知框架 → 核心信念
- 换维度的信号：追问开始重复、用户答案变短、出现"不知道"

## 第二轮深度会话（Q9-Q22）

### 维度轨迹

```
主角起源 ─→ 出丑年代 ─→ 无挫折的矛盾 ─→ 积压型消化 ─→ 内部高标准
  (Q9)        (Q10)        (Q11-Q12)         (Q13)         (Q14)

比较焦虑 ─→ 客观丰足vs主观虚 ─→ 戏台崩塌恐惧 ─→ 夸大的代价
  (Q15)         (Q16)                (Q17)          (Q18)

苗头→80% ─→ 拼凑糊弄分锅 ─→ 模式起源 ─→ 父母的影子 ─→ 垮掉的定义
  (Q19)        (Q20-Q21)        (Q22)       (Q23-Q24)     (Q25-Q26)
```

### 挖掘出的核心机制

#### 1. 苗头→80% 机制

用户坦白：很多事只是"苗头"（种子阶段）就对外宣称"80%"。不是从50%吹到80%，而是从0直接跳到80%。

**后续行为链**：
```
苗头 → 宣称80% → 被自己的宣称绑架 → deadline到
  → 拼凑糊弄 → 继续糊弄 → 找外部原因 → 分锅
```

**诊断价值**：这不是常规的夸大，而是一种**外部动力管理策略**——用公开吹牛来绑架自己行动。

#### 2. "戏台崩塌"恐惧

关键修正：用户最初说"不同社交场合的人凑在一起也特别淡定"（第一版记录错误），随后**修正为"特别担心"**。

**原因**：在每个圈子面前演了不同版本的自己——当两个观众同时出现时，不知道按哪个剧本走。

**与"主角心态"的关系**：主角需要控制叙事——多圈子交汇意味着叙事的控制权被分散。

#### 3. 吹是系统预设，不是环境响应

用户原话"钱少有钱少吹法，钱多就吹得更大一点"揭示核心真相：
- 不是因"不够"而吹（环境驱动）
- 是任何条件下都会自然往上加20%（系统预设）
- 因此"等有钱/成功就不吹了"是伪命题

#### 4. 父母的"低音量版"

模式来自家庭（父母也有类似倾向），但被用户"拧大了音量"——不是对抗形成的，是继承+放大。

### 关键洞察更新

| 之前以为（第一轮） | 现在发现（第二轮） |
|-------------------|-------------------|
| "直觉决策"是行为模式特征 | 直觉是主角叙事的一部分——"我的世界我做主，不需要外部数据验证" |
| 主角心态从外部动作中表现 | 主角心态本质是**有意识的自我欺骗**——知道是假的，但有用 |
| 社交切换高手 | ❌ **戏台崩塌恐惧**——不同圈子交汇=暴露风险 |
| 因条件不足而夸大 | 条件充足时也会夸大——这是**系统预设**，不是环境响应 |
| 模式来自童年/家庭对抗 | 父母"低音量版"→用户拧大音量（继承+放大，非对抗） |

## 跨会话通用提问技巧

### 推荐的提问弧线

```
抽象→具体→分析→经历 → 抽象→具体→分析→经历 ...
```

具体路径：自我定义（抽象）→ 压力来源（行为）→ 决策模式（分析）→ 人生故事（经历）→ 切换维度

### 高价值追问模板

**① "苗头→80%" 追问序列**
当用户承认"夸大"或"吹"时，按此序列下探：
1. 你对外宣称什么程度？实际什么程度？（量化差距）
2. 苗头到宣称之间，你打算怎么填这个差距？（揭示策略）
3. deadline到了差距还在，你怎么办？（暴露兜底机制：拼凑→糊弄→找外部原因→分锅）

**② "垮掉定义" 追问**
当用户说"不吹就会垮"时，问：
- 这个"垮掉"具体指什么？经济？社会评价？自我认知？
- 如果三个都垮，哪个最先发生？
- 如果有一种保障让这三个都不会垮，你还会继续吹吗？（区分"因恐惧而吹"vs"因习惯而吹"）

**③ "戏台崩塌" 诊断**
当用户表现出社交焦虑时，区分两种模式：
- **真·社交切换高手**：不同圈子游刃有余，不焦虑
- **戏台崩塌恐惧**：不同圈子交汇=暴露风险
诊断问题："不同圈子的人凑到一起，你是更兴奋还是更担心？"

**④ "系统 vs 环境" 诊断**
当用户说"有了X我就不吹了"时，追问不同量级下的行为：
- 钱少怎么吹？钱多怎么吹？
- 如果条件好了还是一样吹——说明这是系统预设，不是环境响应

**⑤ "我不知道" 不跳过**
用户说"不知道"时，不是空白而是数据。可能是：
- 从未问过自己这个问题（元认知缺口）
- 有答案但不敢面对（防御机制）
- 需要换一种问法（从抽象→具体）

### 完整画像示例

```

学习模式：   负向过滤（只排除错误，不指向正确）

情绪模式：   积压型消化——小错累积发酵，不爆发但持续消耗
```

### 运作闭环图

```
犯错/力不从心 
  → 扮演主角/夸张/表演乐观 
    → 不同圈子碰面 = 暴露风险 
      → 更焦虑 → 更用力扮演 → 更"虚" → 累积更累
```

HR角色负责：决策机制、职场表现、外部社会角色、成就动机
心理咨询师角色负责：底层信念、情感矛盾、自我欺骗、存在主义议题

切换方式：从业务议题自然过渡到心理议题（如"你说你扮演乐观——那独处时呢？"）

## 记录坑点（汇总）

- **不要在 session 文件中使用 patch 直接替换标题**（容易导致 Q3/Q4 错位）
- 正确方式：将新 Q&A 追加到文件末尾，统一整理时再重建
- questions-log.md 的表格格式容易损坏，推荐用固定宽度 markdown 表格，管道符号严格对齐
- profile.md 的"各类痕迹"改为"跨维度观察"更清晰，含：主线、矛盾点、模式、变化趋势
- **用户原话要保留修正版本**——Q17 用户先说"淡定"后改口"担心"，最初记录错了就要主动修正，不要假装第一版是对的
- **综合报告优先展示"运作闭环图"**——用户对系统化图示的接受度远高于逐点分析
