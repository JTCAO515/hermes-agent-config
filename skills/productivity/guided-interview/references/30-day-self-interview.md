# 30-Day Self-Interview (30天自我访谈报告项目)

> 30天结构化自我访谈项目，每日通过 cron 执行。Agent 扮演存在主义心理咨询师角色，基于设定好的每日主题提问，维持累积式报告文件。用户非技术背景决策者，脾气急，用业务语言描述需求。

## 与标准 Guided Interview 的区别

| 维度 | 标准引导式访谈 | 30天自我访谈 |
|------|---------------|-------------|
| 调度方式 | 用户发起、按需执行 | Cron驱动，每天固定时间执行 |
| 用户在场 | 用户在线并可实时回答 | 用户不在场 — 问题通过 cron 交付，回答在下次运行时读取 |
| 文件结构 | 每次会话一个 session 文件 | 单一累积式 report.md |
| 问题结构 | 自由形态、自适应追问 | 由每日主题/阶段预设，3 个问题由轻到深递进 |
| 分析 | 每次更新 profile.md | 每日小分析 + 累积趋势分析 |
| 交付方式 | 直接对话交付 | 问题作为 cron 输出交付；答案从 report.md 读取（下次运行时） |

## 文件结构

```
~/projects/self-interview/
├── report.md              # 累积报告（每天追加，永不覆盖）
```

无 session 子目录 — 所有内容集中在 report.md 中。

## report.md 每日条目格式

```markdown
## 第 N 天 — YYYY-MM-DD（主题）

### 今天问的 3 个问题

1. **（轻）** ...
2. **（中）** ...
3. **（深）** ...

### 预期分析

**Q1（轻）— 标题**
分析正文...

**Q2（中）— 标题**
分析正文...

**Q3（深）— 标题**
分析正文...

### 用户回答摘要
（待用户回答后补充）

### 今日分析
（待用户回答后补充）

### 截止今天的整体分析
（待用户回答后补充）

---
```

**关键惯例：**
- 已记录的旧回答**永远不要修改**。只追加新内容。
- 「待用户回答后补充」作为占位符保留，直到下次运行时用户已回答。
- 用户回答后：补全「用户回答摘要」「今日分析」「截止今天的整体分析」三个 section。
- 三个 section 要么全部保持占位符，要么全部填写。

## 执行流程

### 当前日期步骤（问问题阶段）

1. **读取 report.md** — 检查从上次运行到现在用户是否有新回答
2. **如果有新回答**（之前标为「待用户回答后补充」的现在有内容了）：补写「今日分析」和「截止今天的整体分析」
3. **读取之前的「截止今天的整体分析」** — 注意是否有提到「今天需要注意的方向」
4. **根据今日主题和累积分析方向，调整今天的问题**
5. **问 3 个问题**（轻→中→深递进）
6. **写预期分析**（解释每个问题想了解什么）
7. **追加 Day N 条目到 report.md**

### 下一次执行步骤（读回答阶段）

1. 读取报告，检查之前的「用户回答摘要」是否已被用户填写
2. 如果是：为那一天写「今日分析」和「截止今天的整体分析」
3. 然后正常推进当天的问题

## 问题设计规则

- **第一个（轻）**：与今日相关，但可回答、不吓人。让用户有进入状态的感觉。
- **第二个（中）**：引用之前回答中的具体内容，建立连续性。让用户感觉到被认真对待。
- **第三个（深）**：触及模式的核心。可以挑战用户，但不评判。

**所有问题必须：**
- 具体（有场景、有时间、有细节）
- 由浅入深
- 只说问题，不给建议、不安慰、不评价
- 用简体中文，语言自然

## 已知陷阱

### ⚠️ 日期编号不一致

脚本输出 `DAY=11` 但 report.md 中的日历编号可能不同（因为有些天跳过了或因 cron 计算方式不同）。**信任脚本的 DAY 值**，同时记录当天的日历日期。

### ⚠️ Day 跳过处理

如果某天 cron 未执行或用户无回答，保留「记录中断」或「未回答」标记。不要自动补写、不要调整后续日期编号。

### ⚠️ 旧数据不动

已经写好的问答内容（包括已填写的「用户回答摘要」）永远不要修改。只需要检查「待用户回答后补充」这个占位符是否已被用户替换为实际内容。

### ⚠️ 三个 section 一致性

「用户回答摘要」「今日分析」「截止今天的整体分析」要么全是占位符，要么全都有内容。不要出现部分填写部分占位符的状态。
