# 合同审阅

> 四阶段流水线：提取 → 拆解 → 分析 → 深读。不同阶段使用不同 skill 的能力。

## 🚨 触发条件 — 必须优先加载，不可绕过

**正确做法（唯一路径）：**
1. 用户触发 ← 识别到关键词
2. 立即 → `skill_view('contract-review')` 【第一条工具调用】
3. 加载完成后 → 输出本 skill 的四阶段流水线概览

用户发送 PDF/Word/图片格式的合同文件，或粘贴合同文本，要求审阅。

## 流水线

### 阶段 1：提取（ocr-and-documents）

- PDF 文本类 → pymupdf 提取全文
- PDF 扫描件 → marker-pdf OCR
- DOCX → python-docx
- **DOC（旧格式 .doc）→ antiword（`antiword -m UTF-8 <file>` 提取纯文本）或 catdoc**
- 图片 → 由用户先转文字

**注意事项：**
- `.doc` 文件是旧版 Word 格式（Composite Document File V2），`python-docx` 不支持，**必须用 `antiword` 提取**
- 安装：`sudo apt-get install -y antiword`
- 提取命令：`antiword -m UTF-8 "合同文件.doc"`
- antiword 输出可能包含 `[pic]` 占位符标记图片位置，不影响文本分析
- 合同通常有页眉页脚编号、表格、签章区。提取时必须保留页码信息，表格数据要核对行列归属（参考 ocr-and-documents 的表格验证协议）。

### 阶段 2：拆解（knowledge-extractor）

从合同全文提取结构化要素：

```
合同概要表：
| 项目 | 内容 |
|------|------|
| 合同编号 | |
```

- 签约地决定了管辖权

### 阶段 3：分析（structured-thinking + 合同知识）

按三级风险评级输出：

**🔴 高风险（必须处理）**
- 对用户有单方面不利的条款
- 与法律强制性规定冲突的条款
- 金额/责任/期限存在重大歧义
- "阴阳条款"——特别约定与正文冲突

**🟡 中风险（建议关注）**
- 逻辑矛盾（如选项全勾"不同意"）
- 未签署/空白的必要附件
- 隐性成本未明示
- 权利不对等

**🟢 低风险 / 提示**
- 管辖权不便
- 单方保留调整权
- 不保证性质的条款（如"不保证同住""不保证同一车厢"）

每条风险标注依据原文的条款号或段落。

## ⚠️ 立场化审阅协议（2026-05-30 新增）

**用户纠正记录：** 本 skill 默认输出平衡/中立的风险分析。但用户明确要求"要从消费者角度去审阅这个合同，最大程度地去保障消费者的权益"。这意味着：

**规则：** 在开始审阅之前，必须先明确两个问题：
1. **用户的立场**：你是哪一方？甲方/乙方/丙方？还是你只是随便看看？
   - 如果用户没有明确说明，默认 **站在消费者/个人用户一方**（因为这是最常见的场景）
2. **审阅的目标**：你只是想了解风险？还是需要谈判策略？还是需要修改建议？

**消费者立场审阅的核心原则：**
- 每一条风险分析的落点是**"对方（公司方）可能怎么利用这个条款对你造成损害"**，而不是笼统的"这个条款不公平"
- 给出**具体可执行的谈判话术**，让用户可以直接用
- 标注哪些是 **底线条款不改就不签**，哪些是 **可以协商** 的
- 计算**经济损失场景**：在什么情况下你会亏多少钱
- 关注**合同中没有写的内容**——公司方的义务空白、服务中断后的补救缺位、退费机制缺失

### 立场化输出示例对比

| 维度 | 平衡分析 | 消费者立场分析 |
|------|----------|----------------|
| 续签条款 | "经甲方同意方可续签，建议三方协商" | "甲方可以单方面不让你续签，你投入的服务费和新人的适应成本就是他们的筹码——签之前改掉" |
| 退费条款 | "应退还剩余服务费，建议明确计算方式" | "退多少钱、多久退、按什么标准——全空着，甲方可以拖你3个月才退一半" |
| 丙方离职 | "由甲方对接安排替换" | "丙方走了你家里带娃断档，甲方只说'对接'不承诺几天补人——这30天你要自己扛" |

### 阶段 4：深读（strategic-reading 理念）

以用户的立场为"透镜"，逐条分析：
- 哪些条款对你有利，为什么
- 哪些条款对你不利，有多大杀伤力
- 哪些条款看起来公平但实际上有坑
- 谈判空间在哪里

## 注意事项

- **不替代律师**：合同审阅不构成法律意见。重大合同建议用户咨询执业律师。
- **引用条款号**：每个判断都要追溯到合同的条款或段落，让用户能自己核对。
- **中国大陆法律优先**：默认以《民法典》《旅游法》《消费者权益保护法》为参考。用户可指定其他法域。
- **特别约定优先扫描**：合同末尾的"特别约定""补充条款""其他约定"是最容易藏坑的地方，必须优先审查并与正文对照。

## 合同类型特有陷阱

### 旅游合同（优先扫描）

审阅旅游合同时，以下三类问题是高发区，必须优先检查。

> 完整案例见: `references/tourism-contract-example.md`（俄罗斯跟团游，含杂费双重损耗、阴阳条款、夫妻分房等教科书级陷阱）

### 1. "杂费"双重损耗
杂费（如 1500 元/人）通常声明"退团不退"，但同时写入"杂费为总团款的一部分，退团时团费+杂费=总团费为基数计算"。这意味着：杂费本身不退，还被纳入扣费基数——**双重吃亏**。必须单独标注并建议用户要求杂费不计入退团扣费基数。

### 2. 特别约定覆盖正文（"阴阳条款"）
合同末尾的"特别约定"如出现"当本条款与正文冲突时，以此条款为准"，且约定了比正文更苛刻的退费标准（如正文阶梯式 5%-70%，特别约定写 8 天内全损），则形成事实上覆盖。必须逐条对比并明确指出冲突。

### 3. 夫妻/伴侣分房
散拼团合同的默认模式是"同性别拼房"，不保证夫妻同住。对情侣/夫妻出行的用户，这是最大体验风险。必须提前识别并建议确认单房差费用。

## 支持的合同类型（按优先级覆盖）

### 1. 旅游合同（出境游/国内游）
- 标准示范文本 + 特别约定阴阳条款审查
- 杂费/小费陷阱（双重损耗：不退+计入扣费基数）
- 退团阶梯扣费 vs 另行约定冲突
- 详见 `references/tourism-contract-example.md`

### 2. 诉讼融资/三方合作协议
- 丙方（资金方）信息完整性 —— 空白即不可执行
- 分配公式歧义：百分号遗漏、未定义变量、逻辑矛盾
- 利益上限锁定（乙方固定额 vs 丙方动态分成）
- 非签约方握有关键权力（如"清华同方扣费"未定义）
- 缺少违约条款 + 争议解决条款
- 详见 `references/litigation-funding.md`

### 3. 服务协议 / 采购合同 / 一般商务合同
- 付款节点与交付物对应关系
- 验收标准是否可客观判断
- 违约责任是否对等
- 知识产权归属

### 4. 家政劳务合同
- 介绍费不可退条款（显失公平审查）
- 人身损害免责条款的《民法典》第497条效力风险
- 违约金对等但实际不对等（乙方擅自离岗追偿难）
- 公司责任缺位（收款vs服务责任）
- 合同性质（劳务vs劳动关系）的模糊性
- 详见 `references/domestic-service-contract.md`

## 反模式

- ❌ 只说"这个条款有问题"而不说为什么有问题、依据在哪里
- ❌ 给模糊建议（"建议谨慎对待"）→ 必须说清楚谨慎什么、怎么做
- ❌ 遗漏金额速算——退费/扣费场景必须给出具体数字
- ❌ 把旅行社的标准文本当作必须接受的条款——标注哪些是可以谈判的
- ❌ 不加载 contract-review skill 而手动拼凑技能——浪费时间且遗漏已有反模式
- ❌ 看到 "legal/" 分类下的技能但不深读就跳过——用户问合同相关就必须检查 legal/ 分类
- ❌ 认为 "组合通用技能" 优于 "使用专用技能"——专用技能的陷阱库无法被临时组合复制
