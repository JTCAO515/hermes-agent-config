## 关键教训

1. **先写VISION再执行** — 规划文档让自动化迭代有方向，不会跑偏
2. **Batching 提高效率** — 小迭代（如POI信息窗/交通切换/截图）可以合并为一次 commit
3. **标记跳过项** — 需要用户提供凭据的步骤（Supabase密码/阿里云Key）在VISION中用 `⏸️` 标记
4. **定期验证部署** — 不要等到攒了一堆commit再验证，可能一次性发现多个问题难以定位
